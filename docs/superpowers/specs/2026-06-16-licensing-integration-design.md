# Cryptographic License Verification Design Spec

This specification outlines the design for integrating an offline-first cryptographic license verification mechanism (inspired by Keygen.sh cryptographic scheme) into Synapse Desktop.

## 1. Goal

Integrate a secure, offline-first licensing mechanism into Synapse Desktop. When unlicensed, the application is locked behind a license activation dialog. Once a valid license is entered, it is verified cryptographically using **Ed25519 signature verification** and saved to the application settings.

## 2. License Key Format

The license key is a structured string of three parts separated by dots (`.`):
`SYNAPSE-KEY.<Payload_Base64>.<Signature_Base64>`

1. **Header Prefix**: `SYNAPSE-KEY`
2. **Payload (Base64 encoded JSON)**:
   ```json
   {
     "license_id": "LIC-100234",
     "email": "user@example.com",
     "expiry_date": "2027-12-31",
     "product": "Synapse Desktop"
   }
   ```
3. **Signature (Base64 encoded Ed25519 Signature)**:
   The Ed25519 signature generated by signing the raw JSON payload with the developer's Private Key.

## 3. Cryptographic Details

- **Algorithm**: Ed25519 (high performance, modern, secure asymmetric key signature)
- **Public Key**: Hardcoded inside the application's license verification adapter.
  ```python
  # Public Key in PEM format (Example Ed25519 public key)
  PUBLIC_KEY_PEM = b"""-----BEGIN PUBLIC KEY-----
  MCowBQYDK2VwAyEAfvU9vWzG3r5P9fI1sS3+p1lJmR0s8yXk2aO4fK+3JtQ=
  -----END PUBLIC KEY-----"""
  ```
- **Verification Flow**:
  1. Parse input key, split by `.`. Validate structure.
  2. Decode `<Payload_Base64>` and parse it as JSON.
  3. Validate expiration date (`expiry_date` >= today's date in UTC/local time).
  4. Validate `product` equals `"Synapse Desktop"`.
  5. Decode `<Signature_Base64>`.
  6. Verify the signature against the raw JSON payload bytes using the hardcoded Ed25519 Public Key.

## 4. Components

### Domain Layer
- **[NEW] `domain/licensing/entities.py`**:
  Defines `LicenseInfo` dataclass:
  ```python
  from dataclasses import dataclass

  @dataclass
  class LicenseInfo:
      license_id: str
      email: str
      expiry_date: str  # YYYY-MM-DD
      product: str
      is_valid: bool = False
      error_message: str = ""
  ```
- **[NEW] `domain/ports/license_service_port.py`**:
  Defines `ILicenseService` port interface:
  ```python
  from abc import ABC, abstractmethod
  from domain.licensing.entities import LicenseInfo

  class ILicenseService(ABC):
      @abstractmethod
      def verify_license_key(self, key_str: str) -> LicenseInfo:
          """Decode, parse, and cryptographically verify a license key string."""
          pass
  ```
- **[MODIFY] `domain/ports/registry.py`**:
  Register the license service in `DomainRegistry`.

### Infrastructure Layer
- **[NEW] `infrastructure/adapters/license_service.py`**:
  Implements `Ed25519LicenseService` using the Python `cryptography` library.
- **[MODIFY] `domain/config/app_settings.py`**:
  Add `license_key: str = ""` to `AppSettings`. Update `to_dict` and `from_dict`.
- **[NEW] `tools/license_generator.py`**:
  Helper script for developers to generate valid signatures and license keys. (Contains both Private Key generation and License signing logic).

### Presentation Layer
- **[NEW] `presentation/widgets/license_dialog.py`**:
  A modal PySide6 `QDialog` named `LicenseActivationDialog` used to request activation.
  - Matches VS Code / JetBrains dark aesthetic.
  - Heading: Gold/amber key icon, "Activate Synapse Desktop".
  - Text Input: `QPlainTextEdit` for easy copying/pasting of the Base64 key string.
  - Status Label: Red label for error reporting (e.g. "Signature verification failed", "License expired").
  - Buttons: "Activate" (Primary Accent), "Exit" (Standard button).
- **[MODIFY] `presentation/views/settings/settings_view_qt.py`**:
  Add a "Licensing" card or section displaying registration details (e.g., licensed email and expiry date).
  Add a "Deactivate" button. Clicking Deactivate shows a confirmation, clears the key from Settings, and prompts a reboot or shows the Activation dialog.

## 5. Startup Flow Integration

Inside [main.py](file:///home/hao/Desktop/labs/Synapse-Desktop/main.py):
1. Initialize `ServiceContainer` and bootstrap Domain Registry.
2. Load settings via `load_app_settings()`.
3. Verify the `license_key` stored in settings using `DomainRegistry.license_service().verify_license_key(key)`.
4. If the key is invalid:
   - Create `QApplication` instance (needed for dialogs).
   - Apply theme style.
   - Instantiation and `exec()` `LicenseActivationDialog`.
   - If user exits the dialog or fails to activate, call `sys.exit(0)`.
   - If user activates successfully, save the key, close the dialog, and proceed to open `SynapseMainWindow`.
5. If the key is valid, proceed directly to open `SynapseMainWindow`.

## 6. Verification and Testing

### Automated Unit Tests
- Create **`tests/test_licensing.py`**:
  - Test validation of a correctly signed valid key.
  - Test validation of an expired key.
  - Test validation of tampered payloads or signature alterations.
  - Test malformed key strings.

### Manual Verification
- Run `./start.sh` or `python main_window.py` without a key, verify the activation dialog appears.
- Run `tools/license_generator.py` to create a key, paste it into the dialog, verify activation succeeds and main window opens.
- Go to Settings tab, check license details, click "Deactivate", and verify the app triggers deactivation.
